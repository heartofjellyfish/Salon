# Salon

**A room that changes every week.** A quiet digital salon — a few large paintings
hung in a calm, naturally-lit room that is rehung once a week. Built to feel like
_entering a room_, not browsing a website.

- **This Week's Room** (`/`) — the room currently on view, with parallax, soft
  light you can change (daylight / evening / spotlight), and a curator's note.
- **Focus viewer** — click any painting to dim the room and bring the work
  forward, with high-resolution zoom & pan. `Esc` closes, `←` / `→` browse.
- **Archive** (`/archive`) — past rooms, kept as they were. `/room/[slug]` opens
  any specific room.

## Run it

```bash
npm install
npm run dev      # http://localhost:3000
npm run build    # production build
npm start        # serve the production build
```

Requires Node 18.18+ (developed on Node 22).

## Swap in your own artwork

Two steps, no code beyond the data file:

1. **Drop the image** into `public/paintings/` (e.g. `public/paintings/my-work.webp`).
   High-res JP/PNG/WebP are all fine — `next/image` generates responsive,
   optimized variants automatically.
2. **Point the data at it** in [`src/data/rooms.ts`](src/data/rooms.ts):

```ts
{
  id: "my-work",
  title: "My Work",
  artist: "Artist Name",
  year: "2026",
  imageUrl: "/paintings/my-work.webp", // the file you just added
  width: 120,   // REAL canvas width in centimeters
  height: 90,   // REAL canvas height in centimeters
  dominantColor: "#cdbb9c", // shown instantly while the image loads
  shortNote: "A line for the museum label.",
  details: [{ label: "Medium", value: "Oil on linen" }],
}
```

`width` / `height` are the **real-world dimensions in centimeters**. They drive
both the aspect ratio and the believable scale on the wall: a 168 cm painting
hangs noticeably larger than an 80 cm one, and all works share a museum eye-line.

The starter ships SVG color-field placeholders generated by
`node scripts/generate-placeholders.mjs`. Once a work has a real image, its
placeholder is unused and can be deleted.

## Add a weekly room

Add a `Room` to the array in [`src/data/rooms.ts`](src/data/rooms.ts). The room
whose `startDate`…`endDate` range contains today is shown on `/` automatically;
the rest fall into the archive. Set a future date range to schedule a room.

```ts
{
  id: "2026-w22",
  slug: "next-week",
  title: "Next Week",
  theme: "A short, evocative subtitle",
  curatorNote: "A few quiet sentences for the wall.",
  startDate: "2026-05-25",
  endDate: "2026-05-31",
  roomStyle: "warm",          // optional palette nudge
  defaultLight: "daylight",   // daylight | evening | spotlight
  paintings: [ /* 3–5 works */ ],
}
```

## How it's built

- **Next.js 15** (App Router) · **TypeScript** · **Tailwind CSS v4** ·
  **Framer Motion** (`motion`).
- The room is **pure-CSS 2.5D** — layered wall / light / floor / plant elements
  with realistic cast shadows and mouse parallax. No 3D engine.
- Light moods are CSS custom properties registered with `@property` so switching
  between daylight, evening, and spotlight interpolates smoothly.
- The zoom/pan viewer ([`ZoomPanImage`](src/components/ZoomPanImage.tsx)) is a
  small dependency-free surface (wheel / pinch / drag / double-click). For true
  gigapixel artwork, swap it for a tiled viewer such as **OpenSeadragon** (with
  DZI tile pyramids) — the surrounding viewer chrome can stay the same.
- Reveal/visibility transitions are CSS-driven; Framer Motion is used only for
  interactive, continuous motion (parallax, hover) so content is never hidden
  behind a paused animation. `prefers-reduced-motion` is respected throughout.

### Project structure

```
src/
  app/                 # routes: / , /archive , /room/[slug]
  components/          # GalleryRoom, RoomWall, PaintingFrame, PaintingLabel,
                       # CuratorNote, PaintingViewer, ZoomPanImage,
                       # WeeklyRoomArchive, RoomNavigation, LightToggle
  data/rooms.ts        # the weekly rooms (edit this)
  lib/                 # types, room lookup helpers, hanging math, date format
public/paintings/      # artwork (placeholder SVGs → your real files)
```
